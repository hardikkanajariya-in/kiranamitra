# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| Latest release | ✅ |
| Older versions | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability in KiranaMitra, please report it responsibly.

**Do NOT open a public issue for security vulnerabilities.**

Instead, email us at: **hkdevs@hardikkanajariya.in**

Please include:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Suggested fix (if any)

We will acknowledge receipt within **48 hours** and provide a detailed response within **7 days**.

## Scope

KiranaMitra is an offline-first mobile application. Security considerations include:

- Local database (WatermelonDB/SQLite) data protection
- Google Drive sync authentication & data encryption
- Bluetooth printer communication security
- MMKV stored credentials and settings

## Best Practices for Users

- Keep your device updated with the latest OS security patches
- Use device-level encryption (enabled by default on modern Android/iOS)
- Protect your Google account with 2FA for Drive sync
- Regularly backup your data via Google Drive sync
